Object method: this
A function that is a property of an object is called its method.
“this” is a placeholder for the object within a function.
let user = {
name: "John",
age: 30,
sayHi() {
alert(this.name);
}
};
user.sayHi(); // John
“this” is not bound. It can be used in any function
its value is evaluated at call-time and does not depend on where
the method was declared, but rather on what object is “before the dot”
methods are functions that are associated within an object.

Chapter 5
Object Literals
An object in JavaScript is a self-contained set of related values and functions
const superman = {
name: 'Superman',
'real name': 'Clark Kent',
height: 75,
weight: 235,
hero: true,
villain: false,
allies: ['Batman','Supergirl','Superboy'],
fly() {
return 'Up, up and away!';
}
};
An object is a grouping of information and/or functions that are relevant together.
To create an object, use curly brackets {}.
You can access an object’s properties by using dot notation or bracket notation
You can call a method (function within an object) by using dot or bracket notation
Superman.fly()
Superman[‘fly’]()
Looping through the properties of an object
for(const key in superman) {
console.log(key + ": " + superman[key]);
}
Adding properties to an object
Superman.city=”Metropolis”
Removing Properties from an object
Delete superman.city
Accessing nested objects
Jla.aquaman.realName
An object literal can be passed as a parameter to a function. This is useful when there
are a large number of parameters, as it allows the arguments to be provided by name and in any order. This means you don't have to remember the order to enter them when invoking a function.

Chap 6: Document Object Model
DOM = Acronym for Document Object Model- represents an HTML document as a network of connected nodes that form a tree-like structure
const listItems = document.getElementsByTagName('li'); -will allow you to get a tag by the tag itself rather than an id.
const heroes = document.getElementsByClassName('hero');- you can access elements by their class name
example.firstChild -returns first child of node (if the parent was a ul, it would be the first in the list)
example.lastChild -returns last child of node
example.nextSibling -returns the next adjacent sibling node (if the node was an li, it would return the next on the list)
example.previousSibling -returns the last adjacent sibling node
example.parentNode -returns the parent node of the queried node
example.nodeValue – returns the text inside the node
example.className can be used to change the class of an element (only use if element has 1 class)
example.classList can be used to add or remove class names to an element (for when there are more than 1 class)
example.classList.contains checks to see if element has a particular class
example.removeChild can be used to remove an element from a page
example.replaceChild can be used to replace an element on a page
The node lists returned by the document.getElementsByClassName() and document.getElementsByTagName() methods arelivecollections
that will update to reflect any changes on the page
Any CSS property names that are separated by dashes must be written in camelCase notation, so the dash is removed and the next
letter is capitalized because dashes are not legal characters in property names
getComputedStyle(example) will return the css styles associated with the object

Chapter 7 Events
The recommended way of dealing with events, and the current standard, is to use event listeners. The screenX and screenY properties show the number of pixels from the left and top of the screen respectively where the event took place.
The clientX and clientY properties show the number of pixels from the left and top of the client that is being used (usually the browser window).
The pageX and pageY properties show the number of pixels from the left and top, respectively, where the event took place in the document.
This property takes account of whether the page has been scrolled
List of types of events at: https://developer.mozilla.org/en-US/docs/Web/Events
Mouse events
Click, mouseup, mousedown, dblclick, mouseover, mouseout, mousemove
(be cautious of using click and dblclick on the same element since the click event will be triggered even if you double click it)
Keyboard events
Keydown (act of pusing a key down), keypress (physical character showing up on the screen), keyup
Touch events
Touchstart, touchend, touchmove, touchenter, touchleave, touchcancel
Bubbling is when the event fires on the element clicked on first, then bubbles up the document tree, firing an event on
each parent element until it reaches the root node.
The bubble phase can be stopped from occurring by adding the event.stopPropagation() method into the callback function.
Capturing starts by firing an event on the root element, then propagates downwards, firing an event on each child element
until it reaches the target element that was clicked on.
